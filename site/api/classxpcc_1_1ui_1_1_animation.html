Detailed Description

template<typename T = uint8_t>
class xpcc::ui::Animation< T >

Be aware that the algortihm for 8bit types is optimized for low computational costs, developed for fast LED fading (

See also
xpcc::ui::Led). Therefore the Animators length is limited to 128ms * value_difference, which is about 32s over the full 8bit range. If you specify a longer time in this case, the Animators simply finishes before. If this is a problem, consider using a 16bit type, which does not have this limitation.
