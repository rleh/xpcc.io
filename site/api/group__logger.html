Detailed Description

The Logger uses a IOStream to read messages and write them to a given "Device".

Write a log message with streams:

XPCC_LOG_DEBUG << "i=" << i << ", y=" << y << xpcc::flush;

The name of the stream represents the log level of the message. Available are:

  • XPCC_LOG_DEBUG
  • XPCC_LOG_INFO
  • XPCC_LOG_WARNING
  • XPCC_LOG_ERROR

TODO check: The flush has to be used, to signal that the message is complete and can be send. A log message can also be generated separately:

int a[3] = {1,2,3};
XPCC_LOG_DEBUG << "a = ";
for(int i = 0; i < sizeof(a); ++i) {
XPCC_LOG_DEBUG << a[i] << ", ";
}
XPCC_LOG_DEBUG << xpcc::flush;

Flow of a call

This is to give an estimation how many resources a call of the logger use. All considerations are for a PC program.

The given call is:

XPCC_LOG_DEBUG << 123456;
  1. The macro resolves to:
    if (XPCC_LOG_LEVEL <= xpcc::log::DEBUG)
    xpcc::log::debug
  2. xpcc::log::debug is an instance of xpcc::Logger:
    Call of xpcc::Logger::operator << (T) (with T = int32_t) which is inline
    • IOStream::operator << (T) (with T = int32_t) is inline
    • IOStream::writeInteger(int32_t)
  3. IOStream::writeInteger(int32_t) will create the formatted string via snprintf()
  4. virtual call of IODevice::write(const char*)
  5. redirect to std::cout

In sum there are two nested method calls with one of them being virtual plus the call of snprintf().

Macro Definition Documentation

#define XPCC_LOG_LEVEL

Define to set the level of the logging to a defined value for each file. To change the logging level in a source file use #undef

1 #undef XPCC_LOG_LEVEL
2 #define XPCC_LOG_LEVEL xpcc::log::INFO

DEBUG < INFO < WARNING < ERROR < DISABLED

#define FILENAME

In contrast to FILE the path to the file is omitted (if this feature is available, otherwise this macro will resolve to FILE).

#define XPCC_FILE_INFO

Will generate something like:

1 "file.cpp (123) >> "

This can be very useful the track the origin of log-messages:

1 XPCC_LOG_DEBUG << XPCC_FILE_INFO
2  << "... something has happened ..."
3  << xpcc::endl;

This will result into:

1 file.cpp (123) >> ... something has happened ...
