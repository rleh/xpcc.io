Detailed Description

A lookup-table should be used to store the alpha corrected PWM values, so that no calculation has to be done at run time.

Here is an example for the STM32F4:

#include <xpcc/ui/led.hpp>
// use a lambda function without a closure, this is equivalent to a normal function pointer
xpcc::ui::Led orange([](uint8_t brightness)
{
Timer4::setCompareValue(2, xpcc::ui::table22_16_256[brightness]);
});
// setup Timer4 etc...

This is just a very, very thin wrapper around xpcc::ui::Animation<uint8_t>. If you need more advanced features, have a look there.
